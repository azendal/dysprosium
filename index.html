Dysprosium
0.1

JavaScript Documentation generator agnostic to the way the code is written. This is because its completely based on documentation blocks

Installation

  gem install Dysprosium

The generator is distributed as a Ruby gem. Why? because it is highly available and its such a cool language

Usage

  cd /path/to/javascript_files
  dy file1.js file2.js

You can create a simbolic link of dy binary to your bin path so you can execute it in this way.

Example

  /**
  A JavasScript documentation generator so we can all write javascript the cool way and document that the easy way
  @class Dysprosium
  @example "Simple Usage"
    Dysprosium.parse(files);
  **/
  Class('Dysprosium')({
      /**
      Parses every documentation block on every file and outputs to html the generated documentation
      @method parse <public>
      @argument files  [Array] an array of files to parse for documentation
      @return (null)
      **/
      parse : function(files){
          //implementation
      }
  });

Documentation

Syntax

Documentation Block

  /**
  documentation here :). yes you can put smileys too
  **/

Documentation Block Content

  /**
  The first thing to do is describe what you are documenting. a good news is that you can use markdown to improve this.
  so lets assume this is a class
  @class Foo
  @example simple example
  @todo maybe improve the example dont you think?
  **/

Tag Syntax

A pretty cool thing is that all tags follow the same syntax so you have to learn less
@tag name_of_tag <flag,flag> [Type|Object|String] ({id : Number, name : String}) and a description to clear things up

Serious syntax my not look so appealing so lets look at some examples: 

  // a class named Foo
  @class Foo
  
  // a module named Bar
  @module Bar
  
  // a simple example
  @example new Foo();
  
  // a named example
  @example "Create a Foo" var foo = new Foo();
  
  // a public attribute named fooInstance of type Foo
  @attribute fooInstance <public> [Foo]
  
  // a required argument of type Object with the structure {width:Number, height:Number}
  @argument config <required> [Object] ({width:Number, height:Number})

Available Tags and common forms

  • @class name
  • @module name
  • @interface name
  • @object name
  • @namespace name
  • @attribute name <flag,flag> [Type|Type] (true)
  • @method name <flag,flag> [Type|Type] (true)
  • @argument name <flag,flag> [Type|Type] (true) 'description
  • @todo 'description
  • @note 'description
  • @example 'description
  • @event click ({target : HTMLElement}) 'description
  • @dispatch click
